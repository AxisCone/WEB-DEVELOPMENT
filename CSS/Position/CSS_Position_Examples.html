🎨 CSS Position Properties

Complete Interactive Examples - Static, Relative, Absolute & Fixed

1️⃣ Position: Static

This is the default position for all HTML elements. Elements with position: static follow the normal document flow and ignore top, right, bottom, left properties.

Example: Normal Flow

DEMO
Box 1 - Static (Normal Flow)
Box 2 - Static (Normal Flow)
Box 3 - Static (Normal Flow)
<div class="static-demo">Box 1</div> <div class="static-demo">Box 2</div> <div class="static-demo">Box 3</div> <style> .static-demo { position: static; /* Default behavior */ background: lightblue; padding: 15px; margin: 10px 0; border: 2px solid blue; } </style>
💡 Note: You rarely set position: static explicitly. Use it to reset an element back to normal flow.

2️⃣ Position: Relative

Positions an element relative to its normal position. It still takes space in the document flow but you can move it using top, right, bottom, left.

Example 1: Relative Box Movement

DEMO
Box 1 - Normal Position
Box 2 - Moved 20px down, 30px right
Box 3 - Still takes original space
<div class="relative-container"> <div class="relative-box">Box 1</div> <div class="relative-box moved">Box 2</div> <div class="relative-box">Box 3</div> </div> <style> .relative-container { position: relative; } .relative-box.moved { position: relative; top: 20px; left: 30px; } </style>

Example 2: Product Card with Badge

DEMO
📦 Product Name 5
🎮 Gaming Console 3
<div class="card"> Product Name <span class="badge">5</span> </div> <style> .card { position: relative; padding: 20px; border: 1px solid #ccc; } .badge { position: relative; top: -8px; display: inline-block; background: #ff6b6b; color: white; border-radius: 50%; width: 25px; height: 25px; } </style>
💡 When to Use: Small nudges, slight adjustments, or as a parent reference for absolutely positioned children.

4️⃣ Position: Fixed

Positions an element relative to the viewport (browser window). It stays in place even when scrolling. Removes element from normal flow.

Example 1: Fixed Navbar

DEMO

There's a fixed navbar at the top of this entire page! Try scrolling.

<nav class="navbar"> <span>Home</span> <span>About</span> <span>Services</span> </nav> <style> .navbar { position: fixed; top: 0; left: 0; right: 0; background: #333; padding: 15px 20px; z-index: 1000; } body { margin-top: 60px; /* Prevent content overlap */ } </style>

Example 2: Scroll to Top Button

DEMO

Scroll down and you'll see a "↑" button in the bottom-right corner. Click it to scroll back to top!

<button class="scroll-to-top" onclick="window.scrollTo(0, 0)">↑</button> <style> .scroll-to-top { position: fixed; bottom: 30px; right: 30px; background: #667eea; color: white; width: 50px; height: 50px; border-radius: 50%; z-index: 999; } </style> <script> window.addEventListener('scroll', function() { const btn = document.querySelector('.scroll-to-top'); if (window.scrollY > 300) { btn.classList.add('show'); } else { btn.classList.remove('show'); } }); </script>

Example 3: Fixed Sidebar Chat Widget

DEMO

Look at the bottom-right corner - there's a fixed chat widget that stays visible while you scroll!

<div class="fixed-sidebar"> <h4>💬 Need Help?</h4> <p>Chat with us anytime!</p> </div> <style> .fixed-sidebar { position: fixed; right: 20px; bottom: 120px; background: #764ba2; padding: 20px; z-index: 998; } </style>

📊 Quick Comparison Table

Property Document Flow Relative To Space Taken Best Use Case
static ✅ Normal N/A ✅ Yes Default, rarely set
relative ✅ Normal Its normal position ✅ Yes Small adjustments, parent ref
absolute ❌ Removed Positioned parent ❌ No Overlays, modals, badges
fixed ❌ Removed Viewport (window) ❌ No Sticky nav, floating buttons

✅ Key Takeaways

1. Static: Default behavior - element follows normal flow
2. Relative: Moves from its own position, stays in flow, acts as reference for absolute children
3. Absolute: Overlays mode, removed from flow, positions relative to nearest positioned parent
4. Fixed: Sticks to viewport/window, stays in place while scrolling
💡 Golden Rule: Use position: relative on parent, position: absolute on child
💡 Always Remember: Use z-index with fixed and absolute positioning to control layering
